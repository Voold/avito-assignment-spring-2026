## ≽ ^⎚ ˕ ⎚^ ≼ Быстрый запуск

> **Важно:** Предварительно запустите Docker Desktop или демон Docker.

1. **Клонирование репозитория:**

   ```bash
   git clone git@github.com:Voold/avito-assignment-spring-2026.git
   cd avito-assignment-spring-2026
   ```

2. **Запуск всей инфраструктуры (Frontend + Backend):**
   ```bash
   docker-compose up --build
   ```

**Доступ к приложению:**

- **Frontend:** http://localhost:5173
- **Backend API:** http://localhost:3000

## .✦ ݁˖ Основные (необязательные) фичи и реализации

- **Docker-native:** Полная поддержка запуска клиента и сервера через `docker-compose`.
- **AI Integration:** Безопасная работа с GigaChat API через самописный прокси-сервер. Для сокрытия Secret Key от клиента и предотвращения утечки токенов.
- **Performance & UX:**
  - Реализовано прерывание (Abort) активных запросов при переходе между страницами (через TanStack Query), что экономит трафик и ресурсы сервера.
  - Использование React Compiler для автоматической оптимизации рендеринга.
  - Реализована система смены темы.

## /•᷅‎‎•᷄\੭ Самостоятельные решения: AI Proxy

Для работы с GigaChat мной был развернут отдельный микросервис-прокси по адресу `https://avito.voold.online`.

- **Безопасность:** Клиент никогда не видит secret key.
- **Токены:** Автоматическое обновление OAuth-токенов Сбера на стороне сервера.

## ฅ^.\_.^ฅ Локальная разработка

Если вы планируете изменять код, установите зависимости локально (рекомендуется `pnpm`):

```bash
pnpm install
```

**Запуск сервисов отдельно:**

- **Backend:**

```bash
  pnpm --filter "./apps/backend" start
```

- **Frontend:**

```bash
  pnpm --filter "./apps/frontend" dev
```

## ദ്ദി◝ ⩊ ◜.ᐟ Оптимизация запросов

В проекте реализован механизм автоматической отмены запросов. Если пользователь инициировал тяжелый запрос к AI или API и перешел на другую страницу, `AbortController` (в связке с TanStack Query) мгновенно обрывает соединение, предотвращая "зависание" ответов в никуда. А также дебаунс на строке поиска, с задержкой 700ms, для лучшей оптимизации нагрузки и UX!

> **Важно:** У меня есть навыки работы в области DevOps, я могу настроить CI/CD и в целом работу сервера, но в данном проекте это было бы избыточно. Т.к. нет даже базовой авторизации - любой может поменять данные. Но навык есть :3
